# Requirement Gathering
## Functional
### Business Context & Value
- What is the problem being solved with this data or pipline.
- What is the current manual or legacy process, and what are its pain points or it is new.
- What is the estimated ROI or value creation of this project?
- How will be know if this project is successfull.Define KPIs
- What is the deadline for the project
### Source Data
- What are source system
- Is this going to be pull or push based approach to get data from source
- Data Refresh schedule at source end
- Incremental/Full Refresh each day
- If Incremental is there way to identify like a column from source or we need to do cdc at our end.
- Any schedule/dependency/event from source system notifying that their processing is done & we are good to consume data will help scheduling our processing.
